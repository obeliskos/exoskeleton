▸ startWatcher(path, eventBaseName)

Starts our container singleton watcher on path specified by user. Events will be emitted as multicast, so if multiple forms load multiple watchers, they should distinguish themselves with the eventBaseName parameter.

startWatcher(path: string, eventBaseName: string)
Parameters
path (string) Path to 'watch'.
eventBaseName (string) Optional base name to emit with.
Example
// The following will generate 'download.created', 'download.deleted', and 'download.changed' events.
exoskeleton.file.startWatcher("C:\downloads", "download");
exoskeleton.events.on("download.created", function(data) {
  // data params is json encoded string (for now).
  exoskeleton.logger.logInfo("download.created", data);
});

▸ start(processStartInfo)

Starts a process resource by providing information in a ProcessStartInfo format.

start(processStartInfo: object)
Parameters
processStartInfo (object) Serialized javascript object closely resembling a c# ProcessStartInfo object.
Name Description
processStartInfo.FileName string? filename of program to load.
processStartInfo.Arguments string? string containing arguments to launch with.
processStartInfo.CreateNoWindow bool? whether to start the process in a new window.
processStartInfo.ErrorDialog bool? whether to show error dialog if process could not be started.
processStartInfo.LoadUserProfile bool? whether the windows user profile is to be loaded from the registry.
processStartInfo.UseShellExecute bool? whether to use the operating system shell to start the process.
processStartInfo.WorkingDirectory bool? working dir of proc when UseShellExecute is false. dir containing process when UseShellExecute is true.
Example
exoskeleton.proc.start({
  FileName: "notepad.exe",
  Arguments: "c:\\docs\\readme.txt"
});

ExoEventEmitter

Event emitter for listening to or emitting events within your page(s)

This is exposed via exoskeleton.events. May be used internally to your container or used to emit or listen to 'multicast' events which are propagated to all host windows.

Event names starting with 'local.' are not to be multicast, otherwise the event will be multicast with an eventname prefixed with 'multicast.'.

new ExoEventEmitter(exo: object, options: object)
Parameters
exo (object) Instance to the exoskeleton com interface
options (object) options to configure event emitter with.
Name Description
options.asyncListeners bool (default false) whether events will be emitted asynchronously.
