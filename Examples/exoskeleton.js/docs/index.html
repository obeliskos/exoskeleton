Exoskeleton.js

Exoskeleton.js is a javscript library which exposes the larger exoskeleton framework for building native windows applications using html/javascript. This exoskeleton.js javascript library is what this documentation will cover.

Navigation

Here are some helper functions for navigation.

Exoskeleton

exoskeleton.js is a wrapper interface for accessing com object functionality exposed by the exoskeleon shell.

▸ convertDateToUnix(dateString)

Converts a .NET date to unix format for use with javascript.

convertDateToUnix(dateString: string): int
Parameters
dateString (string) String representation of a (serialized) .net DateTime object
Returns
int: Number of millseconds since 1/1/1970
Example
// look up some directory info
var dirinfo = exoskeleton.file.getDirectoryInfo("c:\\myfolder");
// convert its last write time to unix (number of ms since 1/1/1970)
var unixTime = exoskeleton.main.convertDateToUnix(dirinfo.LastWriteTimeUtc);
// create a javascript date from unix format
var dt = new Date(unixTime);
▸ formatUnixDate(date, format)

Converts a javascript unix epoch time to a .net formatted date. See ms docs

formatUnixDate(date: (number | Date), format: string): string
Parameters
date ((number | Date)) javascript date object or unix epoch time
format (string) a .net ToString() format string to apply to date
Returns
string: The date formatted to your provided string format
Example
var now = new Date();
// 24 hr date and time
var result = exoskeleton.main.formatUnixDate(now, "MM/dd/yy H:mm:ss");
alert(result);
// formatted date only
result = exoskeleton.main.formatUnixDate(now.getTime(), ""MMMM dd, yyyy");
alert(result);
// formatted time only
result = exoskeleton.main.formatUnixDate(now, "hh:mm:ss tt");

▸ showOpenFileDialog(dialogOptions?)

Allows user to pick a file to 'open' and returns that filename. Although only a few options are documented here, any 'OpenFileDialog' properties may be attempted to be passed.

showOpenFileDialog(dialogOptions: object?)
Parameters
dialogOptions (object?) optional object containing 'OpenFileDialog' properties
Name Description
dialogOptions.Title string? title to display on open file dialog
dialogOptions.InitialDirectory string? initial directory to pick file(s) from
dialogOptions.Filter string? filtering options such as "txt files ( .txt)| .txt|All files ( . )| . "
dialogOptions.FilterIndex int? the index of the filter currently selected in the file dialog box
dialogOptions.Multiselect bool? whether to allow user to select multiple files
Example
// example passing a few (optional) dialog initialization settings
var dialogValues = exoskeleton.main.showOpenFileDialog({
  Title: "Select myapp data file to open",
  InitialDirectory: "c:\\mydatafolder",
  Filter: "dat files (*.dat)|*.dat|All files (*.*)|*.*"
});
// if user did not cancel
if (dialogValues) {
  console.log("selected file (name) : " + dialogValues.FileName);
}

▸ startWatcher(path, eventBaseName)

Starts our container singleton watcher on path specified by user. Events will be emitted as multicast, so if multiple forms load multiple watchers, they should distinguish themselves with the eventBaseName parameter.

startWatcher(path: string, eventBaseName: string)
Parameters
path (string) Path to 'watch'.
eventBaseName (string) Optional base name to emit with.
Example
// The following will generate 'download.created', 'download.deleted', and 'download.changed' events.
exoskeleton.file.startWatcher("C:\downloads", "download");
exoskeleton.events.on("multicast.download.created", function(data) {
  // data params is json encoded string (for now).
  exoskeleton.logger.logInfo("multicast.download.created event received", data);
});

▸ addMenuItem(menuName, menuItemName, emitEventName?)

Adds menu subitems to an existing menu or submenu. Any event emitted on click will be passed the menu item text as parameter.

addMenuItem(menuName: string, menuItemName: string, emitEventName: string?)
Parameters
menuName (string) The text of the parent menu or submenu to add subitem to
menuItemName (string) The text of the new subitem to add
emitEventName (string?) The local event name to unicast when the menu is clicked.
Example
exoskeleton.menu.initialize();
exoskeleton.menu.addMenu("File");
exoskeleton.menu.addMenuItem("File", "Open", "FileOpenEvent");
exoskeleton.menu.addMenuItem("File", "New");
// for this example we will use the same common event name for submenu items
// these can also be different
exoskeleton.menu.addMenuItem("New", ".txt file", "FileNewEvent");
exoskeleton.menu.addMenuItem("New", ".png file", "FileNewEvent");
exoskeleton.on("FileOpenEvent", function() {
  alert('File/Open clicked');
});
exoskeleton.on("FileNewEvent", function(data) {
  alert('File/New/' + data + ' clicked');
});
